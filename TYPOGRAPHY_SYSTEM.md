# Унифицированная система типографики с clamp()

## Обзор

В проекте реализована единая система типографики с использованием CSS `clamp()` для плавного адаптивного масштабирования шрифтов между минимальным и максимальным значением.

## Как это работает

Все стандартные Tailwind классы `text-*` автоматически используют `clamp()` для адаптивного масштабирования:

- **Минимум** — размер на мобильных устройствах (320px viewport)
- **Предпочитаемое значение** — формула с viewport units (vw) для плавного роста
- **Максимум** — размер на больших экранах (1200px+ viewport)

## Рекомендации по использованию

### ✅ Правильно

```html
<!-- Используйте один класс, clamp() сделает адаптивность автоматически -->
<h2 class="text-4xl font-medium">Заголовок</h2>
<p class="text-lg">Текст параграфа</p>
```

### ❌ Неправильно

```html
<!-- НЕ используйте медиа-запросы для размеров шрифтов -->
<h2 class="text-2xl md:text-3xl lg:text-4xl xl:text-5xl">Заголовок</h2>
<!-- clamp() уже делает адаптивность, медиа-запросы не нужны -->
```

### Исключения

Медиа-запросы можно использовать для других свойств (отступы, цвета, layout), но НЕ для размеров шрифтов:

```html
<!-- ✅ Правильно: медиа-запросы для отступов -->
<h2 class="text-4xl mb-8 md:mb-12 lg:mb-16">Заголовок</h2>

<!-- ✅ Правильно: медиа-запросы для layout -->
<div class="text-lg grid grid-cols-1 md:grid-cols-2">...</div>
```

## Базовые HTML элементы

Базовые HTML элементы (`h1-h6`, `body`, `p`) также используют `clamp()` автоматически через глобальные стили в `main.css`:

```html
<!-- Автоматически используют clamp() -->
<h1>Заголовок 1</h1>
<h2>Заголовок 2</h2>
<p>Параграф текста</p>
```

## Миграция существующих компонентов

При обновлении существующих компонентов:

1. **Удалите медиа-запросы для размеров шрифтов:**
   ```html
   <!-- Было -->
   <h2 class="text-2xl md:text-3xl lg:text-4xl xl:text-5xl">Заголовок</h2>
   
   <!-- Стало -->
   <h2 class="text-4xl">Заголовок</h2>
   ```

2. **Выберите средний размер из диапазона:**
   - Если было `text-2xl md:text-3xl lg:text-4xl xl:text-5xl`, используйте `text-4xl`
   - Если было `text-base md:text-lg`, используйте `text-lg`

3. **Сохраните медиа-запросы для других свойств:**
   ```html
   <!-- Отступы, layout и другие свойства остаются с медиа-запросами -->
   <h2 class="text-4xl mb-8 md:mb-12 lg:mb-16">Заголовок</h2>
   ```

## Файлы системы

- `tailwind.config.js` — конфигурация с fontSize для специальных классов
- `assets/css/tailwind.css` — переопределение стандартных Tailwind классов
- `assets/css/main.css` — базовые стили для HTML элементов

## Преимущества

1. **Плавное масштабирование** — шрифты плавно адаптируются к размеру экрана
2. **Меньше кода** — не нужны медиа-запросы для каждого размера шрифта
3. **Единообразие** — все компоненты используют одну систему
4. **Производительность** — меньше CSS правил, быстрее рендеринг
5. **Поддержка** — легко обновлять и поддерживать
